# Labeling Rules

이 문서는 평가셋과 이후 파인튜닝 골드셋에 공통으로 사용하는 라벨링 기준표입니다.

## 1. 목적

- `summary`, `focusLabel`, `keywords`, `quality`를 일관된 기준으로 정답화합니다.
- 그룹 제목으로 올려도 되는 표현과 버려야 하는 표현을 명확히 나눕니다.

## 2. 출력 항목

### summary

- 실제 업무를 식별할 수 있는 짧은 한국어 구문
- 48자 이하
- 직무명 반복 금지
- 회사 소개, 복지, 채용 안내 금지

좋은 예시

- `멀티모달 검색 파이프라인 구축과 모델 미세조정`
- `NPU 검증용 테스트벤치 개발과 성능 확인`

나쁜 예시

- `인공지능 엔지니어`
- `연구 지원 프로그램`
- `채용 문의 안내`

### focusLabel

- 그룹 근거로 쓸 수 있는 짧은 핵심 명사구 1개
- 스택, 도메인, 시스템, 검증 대상, 업무 유형 중 하나여야 함

좋은 예시

- `데이터 파이프라인`
- `모델 서빙`
- `컴퓨터비전`
- `검색 시스템`

나쁜 예시

- `위한`
- `또는`
- `석사`
- `학력`
- `제품`
- `서비스`

### keywords

- 2~5개
- 짧은 명사구
- 그룹 제목 seed로 써도 되는 표현만 허용

허용 예시

- `LLM`
- `RAG`
- `PyTorch`
- `컴퓨터비전`
- `데이터 파이프라인`
- `모델 서빙`
- `실험 설계`
- `헬스케어`

금지 예시

- 조사/접속어: `위한`, `또는`, `대한`, `통한`
- 학력/경력 표현: `학력`, `학사`, `석사`, `박사`, `경력`
- 포괄 명사: `제품`, `서비스`, `기술`, `업무`
- 문장형 표현: `인공지능 또는 플랫폼 제품 출시 경험`
- 꼬리 표현: `분석하고`, `정의하고`, `필요한`

### quality

- `high`
  - summary와 keywords 모두 바로 게시 가능
- `medium`
  - 일부는 쓸 수 있지만 표현이 다소 넓거나 약함
- `low`
  - 근거가 약하거나 잡음이 많아 신뢰하기 어려움

## 3. field별 우선 기준

### 상세본문

- 역할 + 실제 업무 신호 우선
- 설명성 문장보다 반복되는 업무 단위를 뽑는다

### 주요업무

- 실제 수행 행위와 시스템 단위 우선
- 예: `모델 서빙`, `데이터 파이프라인`, `실험 설계`

### 자격요건

- 학력/경력보다 역량/스택 중심으로 읽는다
- 예: `PyTorch`, `LLM`, `쿠버네티스`, `통계 분석`

### 우대사항

- 차별화 신호가 있는 항목만 남긴다
- 예: `로보틱스`, `헬스케어`, `클라우드`

### 핵심기술

- 실제 기술명과 스택 중심
- 여기서는 `Python`, `SQL`, `AWS` 같은 기술도 허용한다

## 4. 판정 우선순위

1. 그룹 제목으로 써도 되는가
2. field 의미와 맞는가
3. 다른 공고와 비교 가능한가
4. 문장 조각이 아닌가
5. 사람이 봤을 때 바로 납득되는가
